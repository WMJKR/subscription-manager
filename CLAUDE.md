## 2026-07-11 (차별화 기능 4종 추가)

### 완료한 작업 요약
- **사용량 체크인 → 회당 비용** (`/report`): 구독별로 "이번 주 몇 번 쓰셨나요?" 원탭 질문(안 씀/1~2번/3번 이상)을 표시하고, 응답에 따라 회당 비용을 계산(1~2번→대표값 1.5회, 3번 이상→대표값 4회, 안 씀→경고 문구만 표시). 응답은 `usageCheckIn: { weekKey, frequency }`로 저장하고 ISO 주차(`getWeekKey`)가 바뀌면 자동으로 "체크인 필요" 상태로 리셋됨(히스토리 누적 없음)
- **가격 인상 감지** (`/report`): `Subscription.initialAmount`(최초 등록 금액)와 `amount`(현재 금액)를 분리 관리. `/report`의 "금액 수정"으로 `amount`만 갱신 가능하며, `amount > initialAmount`인 구독만 "OO원(OO%) 올랐어요" 배지로 노출 (변동 없는 구독은 표시 안 함)
- **아이콘 클릭 → 해지/결제관리 딥링크**: 대시보드·알림 화면의 서비스 아이콘을 별도 `<button>`으로 분리해 클릭 시 새 탭으로 이동 (`src/lib/deeplinks.ts`). 프리셋 서비스 중 확실한 공식 도메인만 하드코딩(넷플릭스/유튜브 프리미엄/디즈니플러스/왓챠/웨이브/티빙/스포티파이/멜론/애플뮤직/쿠팡와우), 매핑이 없는 서비스(헬스장·정기배송·직접 입력 등)는 `google.com/search?q={서비스명}+해지방법`으로 폴백. 아이콘 버튼에 `stopPropagation`을 걸어 카드 자체의 클릭(알림 화면의 유지/해지 모달)과 충돌하지 않도록 처리 — 버튼-안-버튼 중첩을 피하려고 `SubscriptionCard`의 바깥 wrapper를 `<button>`에서 `<div role="button">`으로 변경
- **공유/분담 구독**: `/register`에 "다른 사람과 나눠 쓰나요?" 토글 + 인원수 입력 추가, `sharedCount` 필드로 저장(기본 1). **실제 부담액**(`getActualAmount` = 결제금액 ÷ sharedCount, 반올림)을 도입해 이번 달 총 지출액·연간 환산·Top3·카테고리 비중 4개 계산 함수 전부를 raw amount 기준에서 실제 부담액 기준으로 전환. 구독 카드에는 전체 결제금액과 "N명 분담 · 내 부담 O원"을 함께 표시
- **데이터 모델**: `Subscription`에 `initialAmount`, `sharedCount`, `usageCheckIn?` 추가. `storage.ts`의 `normalizeSubscription`에 기존 데이터 방어 로직 확장 — `sharedCount` 없으면 1, `initialAmount` 없으면 현재 `amount`로 간주(즉 과거 데이터는 가격 인상 없음으로 처리됨)
- 새 라이브러리 추가 없음
- 헤드리스 브라우저(Playwright)로 재검증: 분담 구독 등록 → 대시보드 총지출/연간환산/Top3/카테고리 비중이 실제 부담액 기준으로 정확히 반영, 아이콘 클릭 시 올바른 URL로 새 탭 오픈(매핑 서비스/폴백 서비스 모두), 알림 화면에서 아이콘 클릭이 카드 클릭(모달)을 트리거하지 않음, `/report` 체크인 응답별 회당 비용 계산과 "안 씀" 경고 문구, 금액 수정 후 가격 인상 배지와 대시보드 재계산까지 숫자 직접 검산 완료. 콘솔 에러 없음. 375px 레이아웃 확인 완료 (4탭 하단 네비게이션 포함)
